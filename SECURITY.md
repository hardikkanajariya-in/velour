# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

### How to Report

Send an email to **hardik@hardikkanajariya.in** with:

- A description of the vulnerability
- Steps to reproduce the issue
- Any potential impact assessment
- Your suggested fix (if applicable)

### Response Timeline

| Action                    | Timeframe        |
| ------------------------- | ---------------- |
| Acknowledgment            | Within 48 hours  |
| Initial assessment        | Within 5 days    |
| Fix & patch release       | Within 14 days   |
| Public disclosure (if any)| After patch ships |

### Scope

This policy applies to:
- The VELOUR application source code
- Dependencies directly managed by this project
- Configuration files and deployment scripts

### Out of Scope

- Third-party services (Neon, Cloudinary, Razorpay, Resend, etc.)
- Issues in upstream dependencies (report those to the respective maintainers)

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |
| < 1.0   | ❌        |
